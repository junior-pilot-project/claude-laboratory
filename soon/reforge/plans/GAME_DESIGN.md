# 장비 강화 게임 (Reforge) 구현 계획

## Context

장비 아이템(칼/방패)을 강화해 합산 +40 달성을 목표로 하는 웹 기반 게임.  
강화 성공 확률이 떨어지는 로그라이크 스타일의 도박+강화 루프를 구현한다.  
로그인 없이 로컬스토리지로 저장, 게임 종료 시 랭킹 화면으로 이동.

---

## 확정된 비즈니스 로직

### 아이템 등급 및 강화 확률

```
확률 테이블: [100, 90, 80, 70, 60, 50, 40, 30, 20, 10, 5, 1]
                                                          ↑ 이후 계속 1% 유지

등급별 시작 인덱스:
- 상급 (high): index 0 → 첫 강화 100%
- 중급 (mid):  index 4 → 첫 강화 60%
- 하급 (low):  index 9 → 첫 강화 10%

getProbability(grade, currentLevel):
  index = gradeStartIndex + currentLevel
  return index >= 12 ? 1 : PROB_TABLE[index]
```

### 강화 실패 처리
- 실패해도 현재 강화 수치 유지 (다운그레이드 없음)

### 장비 슬롯
- 칼 슬롯 1개 + 방패 슬롯 1개 동시 착용 가능
- 게임 종료 조건: 칼 강화치 + 방패 강화치 ≥ 40

### 강화 비용 (제안 - 인터뷰로 조정 예정)

```
비용 산출 기준:
- 현재 강화 확률 > 30% 구간: costBase + level × 10,000원
- 현재 강화 확률 ≤ 30% 구간: costBase + level × 1,000,000원

제안값:
- 상급 칼/방패 (100% 시작):
  level 0→1(100%): 10,000 / level 6→7(40%): 70,000
  level 7→8(30%): 1,070,000 / level 8→9(20%): 2,070,000 ...

- 중급 칼/방패 (60% 시작):
  level 0→1(60%): 10,000 / level 2→3(40%): 30,000
  level 3→4(30%): 1,030,000 ...

- 하급 칼/방패 (10% 시작):
  level 0→1(10%): 이미 ≤30% 구간 → 1,000,000원부터
```

> ⚠️ 실제 밸런스는 플레이 테스트 후 `config.js` 상수로 조정

### 아이템 구매 가격 (제안)

| 아이템 | 가격 |
|--------|------|
| 하급 칼/방패 | 50,000원 |
| 중급 칼/방패 | 500,000원 |
| 상급 칼/방패 | 5,000,000원 |

### 도박장 (랜덤박스)

보상 분포: `Math.pow(Math.random(), 2) * max` (높은 값일수록 낮은 확률)

| 박스 | 가격 | 보상 범위 | 기대값(약) |
|------|------|-----------|------------|
| 무료 박스 | 0원 | 0~100원 | ~33원 |
| 천원 박스 | 1,000원 | 0~10,000원 | ~3,333원 |
| 만원 박스 | 10,000원 | 0~100,000원 | ~33,333원 |

- 무료 박스: 최대 10개 한 번에 열기 가능, 약 3초 애니메이션
- 오픈 연출: 보물 상자 흔들림 → 뚜껑 열림 → 빛 효과 → 결과 표시

### 랭킹

- localStorage에 로컬 저장 (상위 10개)
- 플레이타임 기준 오름차순 (빠를수록 높은 순위)

## 사운드 설계

Web Audio API로 모든 사운드를 **코드로 생성** (외부 파일 없음, 오프라인 동작).

| 이벤트 | 사운드 설명 | 구현 방법 |
|--------|-------------|-----------|
| 버튼 클릭 | 짧은 클릭음 | 20ms 짧은 화이트노이즈 burst |
| 강화 시도 (슬롯머신 느낌) | 두근거리는 긴장감 | 낮은 주파수 sin파 0.3초 |
| 강화 성공 (일반) | 맑은 상승 차임 | C→E→G 3음 순차 재생 (각 100ms) |
| 강화 성공 (확률 30% 이하) | 더 화려한 팡파레 | C→E→G→C(옥타브) + 에코 효과 |
| 강화 실패 | 둔탁한 실패음 | 하강하는 사각파 + 노이즈 0.4초 |
| 골드 부족 | 에러 버저 | 낮은 buzz 2회 반복 |
| 랜덤박스 흔들림 (3초) | 상자 덜컹거리는 소리 | 반복 노이즈 burst (인터벌 0.2초) |
| 랜덤박스 오픈 | 반짝이는 팡파레 | 상승 글리산도 + 반짝임 노이즈 |
| 골드 획득 | 코인 소리 | 고음 sine 700Hz → 900Hz, 80ms |
| 아이템 구매 | 구매 완료음 | G→C 2음, 부드러운 사인파 |
| 게임 종료 (달성!) | 승리 팡파레 | C-E-G-C-E-G-C 7음 순차 + 리버브 |
| 탭 전환 | 가벼운 whoosh | 200→400Hz 사인파 슬라이드 0.1초 |

### SoundManager 구현 방식

```javascript
class SoundManager {
  constructor() {
    this.ctx = null;          // AudioContext (최초 클릭 시 초기화)
    this.enabled = true;
  }

  init() {
    if (!this.ctx) this.ctx = new AudioContext();
  }

  _play(frequency, type, duration, gainValue = 0.3) { ... }
  _playSequence(notes, interval) { ... }  // 멜로디 순차 재생
  _playNoise(duration) { ... }             // 화이트노이즈

  playClick() { ... }
  playEnhanceReady() { ... }
  playSuccess(isRare = false) { ... }
  playFail() { ... }
  playInsufficientGold() { ... }
  playBoxShake() { ... }
  playBoxOpen() { ... }
  playCoin() { ... }
  playPurchase() { ... }
  playVictory() { ... }
  playTabSwitch() { ... }

  toggle() { this.enabled = !this.enabled; }
}
```

> AudioContext는 브라우저 정책상 사용자 첫 클릭 이후 초기화.  
> 우측 상단 🔊 음소거 버튼으로 on/off 가능.

## 핵심 구현 세부사항

### localStorage 스키마

```json
{
  "gold": 0,
  "startTime": 1749200000000,
  "equippedSword": { "id": "uuid", "grade": "high", "enhancement": 5 },
  "equippedShield": null,
  "inventory": [{ "id": "uuid", "type": "shield", "grade": "mid", "enhancement": 2 }],
  "rankings": [
    { "playTime": 1122000, "date": "2026-06-06" }
  ]
}
```

### 강화 시도 흐름

```
1. 비용 확인 (gold 충분한지)
2. gold 차감
3. Math.random() × 100 < probability → 성공
4. 성공: enhancement++
5. 합계 계산 → 40 이상이면 게임 종료 처리
6. localStorage 저장
7. UI 업데이트
```

### 랜덤박스 오픈 흐름 (최대 10개)

```
1. 수량 선택 (1~10)
2. 비용 차감 (무료=0, 1k/10k박스는 수량×가격)
3. 애니메이션 시작 (CSS: shake → open → shine)
4. 3초 후 결과 표시 (각 박스별 보상 금액)
5. gold 합산
6. localStorage 저장
```

## 구현 순서

1. **config.js** - 모든 상수 정의 (확률, 비용, 아이템 가격, 박스 설정)
2. **gameState.js** - 상태 초기화, localStorage 읽기/쓰기, 게임 종료 감지
3. **itemSystem.js** - 강화 확률 계산, 비용 계산, 강화 시도 함수
4. **gamblingSystem.js** - 박스 보상 계산, 애니메이션 타이밍
5. **rankingSystem.js** - 랭킹 CRUD, 순위 계산
6. **soundManager.js** - Web Audio API 기반 전체 사운드 구현
7. **uiManager.js** - 전체 화면 렌더링, 탭 전환, 강화 결과 연출 (사운드 연동)
8. **index.html + style.css** - 다크 판타지 테마 레이아웃 (🔊 음소거 버튼 포함)
9. **main.js** - 초기화 + 이벤트 바인딩

## 검증 방법

1. 브라우저에서 `index.html` 직접 열기 (서버 불필요)
2. 무료 박스 × 10 → 골드 획득 확인
3. 상점에서 아이템 구매 → 인벤토리 확인
4. 강화 시도 → 성공/실패 결과 + 골드 차감 확인
5. 확률 30% 구간 진입 시 비용 급증 확인
6. 강화합 40 달성 → 랭킹 화면 전환 확인
7. 페이지 새로고침 후 → 이전 상태 복원 확인 (localStorage)
8. 랭킹 여러 번 등록 → 정렬 순서 확인
