# SeatFinder 요구사항 명세

## 개요

버스(고속/시외) 및 기차(KTX) 예매 사이트에서 취소표가 발생하면 자동으로 감지하여 알림을 보내주는 모니터링 도구.

---

## 배경

인기 노선은 예매가 빠르게 마감됨. 취소표를 잡으려면 수동으로 계속 새로고침해야 하는 불편함이 있음. 이를 자동화하여 취소표 발생 즉시 알림을 받고 직접 예매할 수 있게 함.

---

## 지원 사이트 (우선순위 순)

| 순서 | 사이트 | URL |
|------|--------|-----|
| 1 | 고속버스 통합예매 | kobus.co.kr |
| 2 | 시외버스 통합예매 | bustago.or.kr |
| 3 | KTX (코레일) | korail.com |

---

## 기능 요구사항

### 핵심 기능

- [ ] 지정한 노선(출발지/도착지/날짜/시간대)의 잔여석을 주기적으로 확인
- [ ] 잔여석 발생 시 즉시 알림 발송
- [ ] 여러 노선 동시 모니터링 가능

### 알림

- [ ] **텔레그램 Bot 알림** — 어디서든 폰으로 수신
- [ ] **윈도우 토스트 알림** — 컴퓨터 우측 하단 팝업

### 설정

- [ ] `config.json`으로 노선/시간대/간격 설정
- [ ] 체크 간격 최소 1분, 자유롭게 지정 가능
- [ ] 텔레그램 토큰/채팅ID는 `.env` 파일로 분리 (git 제외)

---

## 비기능 요구사항

- 자동 예매 기능 없음 (사이트 보안상 불가)
- 브라우저 크래시 시 자동 재시작
- 로그는 터미널에 출력

---

## 기술 스택

| 항목 | 선택 |
|------|------|
| 런타임 | Node.js |
| 브라우저 자동화 | Playwright |
| 텔레그램 알림 | Telegram Bot API (HTTP) |
| 윈도우 알림 | node-notifier 또는 윈도우 내장 PowerShell toast |
| 설정 | config.json + .env |
