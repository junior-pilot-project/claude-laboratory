# 04 · grill-me 스킬

## grill-me란?

> **"계획이나 설계에 대해 공유된 이해에 도달할 때까지 AI가 집요하게 인터뷰하는 스킬"**

ex-Vercel 엔지니어 **Matt Pocock**이 만든 3줄짜리 스킬. 단순하지만 AI 코딩에서 가장 흔한 실패 원인인 **"요구사항 불일치"**를 코딩 시작 전에 제거한다. GitHub에서 빠르게 바이럴되어 현재 가장 널리 쓰이는 Claude Code 스킬 중 하나다.

**스킬 추가 방법:** 아래 링크 내용 복사 후 Claude에게 스킬 추가 요청  
https://github.com/mattpocock/skills/blob/main/skills/productivity/grill-me/SKILL.md

---

## 핵심 원리

grill-me의 본질은 딱 세 줄이다:

```
계획이나 설계의 모든 측면에 대해 공유된 이해에 도달할 때까지 나를 집요하게 인터뷰해라.
의사결정 트리의 각 분기를 하나씩 내려가며 의존성을 해결해라.
질문에 코드베이스를 탐색해서 답할 수 있다면, 직접 탐색해라.
각 질문에 대해 추천 답변을 제시해라.
```

마지막 줄 — **"추천 답변을 제시해라"** — 가 핵심이다. AI가 질문만 던지는 게 아니라 권장 답변까지 같이 제시하므로, 사용자는 긴 설명 없이 "응" 또는 수정만 하면 된다. 세션이 훨씬 빨라지는 이유다.

---

## 설치 및 사용법

```bash
# 설치 (프로젝트별로 한 번)
/plugin install grill-me@claude-plugins-official

# 새 기능 구현 전 매번 실행
/grill-me
```

설치 후 `.claude/skills/` 디렉토리에 저장되며, 해당 프로젝트의 모든 Claude Code 세션에서 사용 가능하다.

## Wave 구조 — 3단계 질문 파도

### Wave 1 (3~5개 질문)
**기초 파악** — 목표, 컨텍스트, 제약조건

> "이 기능을 쓰는 주요 사용자는 누구인가요?"  
> "기존 인증 로직과 어떻게 연동되나요?"

### Wave 2 (2~4개 질문)
**심화 명확화** — 엣지 케이스, 충돌, 의존성

> "로그인하지 않은 사용자가 이 엔드포인트에 접근하면 어떻게 처리하나요?"  
> "기존 캐시 레이어와 충돌 가능성은 없나요?"

### Wave 3+ (1~3개 질문)
**심층 검토** — 모순, 암묵적 가정, 맹점

> "스펙에서 성능 요구사항을 명시하지 않았는데, 이게 의도적인 건가요?"

각 wave 사이에 Claude가 **사실 · 가정 · 리스크 요약**을 제공한다.

## 슈퍼파워 `/brainstorm`과의 차이

|  | **grill-me** | **슈퍼파워 /brainstorm** |
| --- | --- | --- |
| **질문 주도권** | 사용자의 생각을 파냄 | Claude가 설계 방향 주도 |
| **목적** | 내가 뭘 원하는지 명확화 | 뭘 만들지 어느 정도 아는 상태에서 빠른 설계 확정 |
| **맹점 발굴** | ✅ 강점 | 상대적으로 약함 |
| **세션 길이** | ~45분 심층 인터뷰 | 더 빠른 수렴 |
| **추천 답변** | ✅ 제시함 | 상황에 따라 다름 |

## 언제 쓰나?

- 새 기능 구현 **전 매번** (특히 요구사항이 아직 모호할 때)
- 기술 스택이나 아키텍처 결정을 내리기 전
- 코딩 세션이 자꾸 엉뚱한 방향으로 가는 패턴이 반복될 때
- 랄프루프 전 `--completion-promise`를 정확히 정의하고 싶을 때

> 💡 **핵심 교훈**: AI 코딩 실패의 대부분은 모델이 코드를 못 짜서가 아니라, **요구사항이 끝까지 명세되지 않았기 때문**이다. grill-me는 그 명세 과정을 구조화된 협업 단계로 만든다.
