# 05 · 추천 워크플로우 — grill-me → 스펙 → 구현 → 랄프루프

## 왜 이 순서인가?

> **"AI가 모호함을 채우게 두지 말고, 내가 먼저 모호함을 없애라"**

슈퍼파워 + 랄프루프를 바로 돌리는 건 강력하지만, 스펙이 느슨하면 AI가 완료 조건을 스스로 해석해버린다. grill-me로 스펙을 먼저 갈고닦으면 랄프루프의 `--completion-promise`를 정확하게 정의할 수 있고, 비용 대비 효율이 크게 올라간다.

---

## 4단계 워크플로우

### 1단계 — grill-me × n 세션

**목적**: 내가 진짜 뭘 원하는지 명확하게 하기

슈퍼파워의 `/brainstorm`은 **Claude가 질문을 주도** — AI가 중요하다고 생각하는 방향으로 흐른다.  
grill-me는 **사용자의 생각을 끝까지 파내는** 소크라테스식 구조 — 내가 명시하지 않은 가정과 맹점까지 드러난다.

**새 세션마다 반복하는 이유**: 컨텍스트가 리셋되면서 이전 대화에 묻힌 가정 없이 **fresh eye**로 스펙을 다시 검토하게 된다.

grill-me 동작 방식:
- **Wave 1** (3~5개): 목표, 컨텍스트, 제약조건 파악
- **Wave 2** (2~4개): 엣지 케이스, 충돌, 의존성 명확화
- **Wave 3+** (1~3개): 모순, 암묵적 가정, 맹점 발굴
- 각 wave 사이에 Claude가 사실·가정·리스크 요약 제공

```bash
# 설치
/plugin install grill-me@claude-plugins-official

# 사용 — 새 기능 구현 전 매번 실행
/grill-me
```

> 💡 세션마다 스펙 문서(`specs/feature.md`)를 업데이트하며 누적한다.

---

### 2단계 — 스펙 확정

**목적**: grill-me 세션들의 결과를 하나의 명확한 문서로 수렴

확정된 스펙에 반드시 포함할 것:
- 완료 조건 (랄프루프의 `--completion-promise`로 쓸 문구)
- 테스트 커맨드 (`test_command`)
- 건드리지 않을 범위 (랄프루프가 이상한 방향으로 가는 것 방지)
- 우선순위가 있는 구현 순서

```markdown
<!-- specs/feature.md 예시 -->
## 완료 조건
모든 유닛 테스트 통과 + 타입 체크 통과 시 "DONE" 출력

## 테스트 커맨드
npm test && tsc --noEmit

## 범위 외
- 인증 로직 수정 금지
- DB 스키마 변경 금지
```

---

### 3단계 — 구현

**목적**: 확정된 스펙 기반으로 Claude Code가 코드 작성

- `CLAUDE.md`에 스펙 파일 경로 명시
- 슈퍼파워의 `/execute-plan`을 여기서 활용 가능 (선택)
- 스펙이 기준점이 되므로 중간에 방향이 틀어져도 **복구가 쉽다**

---

### 4단계 — 랄프루프

**목적**: 완료 조건이 충족될 때까지 자동 반복

```bash
/ralph-wiggum:ralph-loop "$(cat specs/feature.md)" \
  --max-iterations 30 \
  --completion-promise "DONE"
```

스펙이 명확하므로:
- `--completion-promise`를 **정확하게** 정의 가능
- 각 iteration마다 AI의 자의적 판단 여지가 최소화
- 루프가 이상한 방향으로 가도 스펙이 기준점으로 작동

---

## 슈퍼파워와의 비교

|  | **슈퍼파워 단독** | **grill-me → 랄프루프** |
| --- | --- | --- |
| **스펙 주도권** | Claude가 질문 주도 | 사용자가 주도 |
| **적합한 상황** | 뭘 만들지 어느 정도 아는 상태 | 요구사항이 아직 모호한 상태 |
| **스펙 품질** | AI가 모호함을 채움 | 모호함을 먼저 제거 |
| **비용 효율** | 스펙 오류 시 비용 큼 | 스펙 투자로 루프 낭비 최소화 |
| **슈퍼파워 활용** | 전체 워크플로우 | `/execute-plan`만 선택 차용 가능 |

> 💡 슈퍼파워를 아예 안 쓰는 게 아니라, **스펙 확정 이후** `/execute-plan`만 빌려 쓰는 방식도 유효하다.

---

## 핵심 교훈

> **랄프루프는 반복 엔진, grill-me는 방향 설정 도구.**
>
> 방향 없이 반복만 돌리는 건 엔진 켜놓고 핸들 안 잡는 것과 같다.
